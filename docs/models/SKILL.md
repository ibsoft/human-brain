# Human-Brain Models Skill

Use this skill when changing, debugging, or operating Human-Brain embedding models, vision models, or model-backed memory behavior.

## Scope

This skill covers:

- Embedding model selection
- Ollama embedding models
- SentenceTransformers embedding models
- Deterministic hash fallback embeddings
- YOLO vision model selection
- Model settings on the Settings page
- FAISS rebuild requirements after changing embedding models
- Safe local-first model operation

## Model Settings

All operational model settings should be changed through the Settings page when possible.

Important settings:

- `embedding_model`
- `embedding_models`
- `ollama_base_url`
- `yolo_model`
- `yolo_device`
- `vision_models`
- `vision_backend`
- `camera_enabled`
- `snapshot_storage_enabled`
- `vision_auto_save`

`.env` is used for deployment defaults and first boot values. Runtime changes belong in Settings.

## Embedding Model Options

Supported embedding model formats:

```text
sentence-transformers/all-MiniLM-L6-v2
sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
ollama:paraphrase-multilingual
ollama:nomic-embed-text
hash
```

### SentenceTransformers

Use `sentence-transformers/<model-name>` for local Python embedding models.

Example:

```text
sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
```

The app removes the `sentence-transformers/` prefix before loading the model through `SentenceTransformer`.

Production note: pre-download models during deployment so the app does not attempt network downloads at request time.

### Ollama

Use `ollama:<model-name>` for local Ollama embeddings.

Example:

```text
ollama:paraphrase-multilingual
```

Ollama must already be running and the model must exist locally.

Default base URL:

```text
http://localhost:11434
```

If Ollama fails, Human-Brain logs a warning and falls back to deterministic hash embeddings so memory writes and context building do not crash.

### Hash

Use:

```text
hash
```

This produces deterministic local vectors without external dependencies. It is useful for tests, offline development, and emergency fallback. It is not semantically strong and should not be used for high-quality production retrieval.

## Changing Embedding Models

Changing `embedding_model` changes how new query vectors and rebuilt memory vectors are produced.

After changing the embedding model:

```bash
python manage.py rebuild-index
```

Why:

- FAISS indexes store vectors generated by the previous model.
- Query vectors from a new model are not comparable to vectors from the old model.
- Rebuilding keeps semantic search and context builder ranking coherent.

## Vision Model Options

Vision uses Ultralytics YOLO through the Vision module.

Default initial model:

```text
models/yolov8n.pt
```

Other examples:

```text
yolov8n.pt
yolov8s.pt
yolov8m.pt
yolov8l.pt
yolov8x.pt
/absolute/path/to/custom.pt
```

Add available models to `vision_models`, one per line, and set the active model in `yolo_model`.
Set `yolo_device` to `cpu`, `cuda`, `cuda:0`, `cuda:1`, or `mps` depending on the host hardware and PyTorch build.

Download missing standard YOLOv8 model files into `./models` with:

```bash
python scripts/download_yolo_models.py
```

Use `python scripts/download_yolo_models.py yolov8n.pt` to fetch only the default lightweight model.

## Camera and Privacy Rules

Camera use must be explicitly enabled.

Default behavior:

- Camera disabled.
- Snapshot storage disabled.
- Auto-save disabled.
- Vision stores metadata only unless configured otherwise.

If snapshot storage is enabled, uploaded/generated snapshots are local files and must be included in backup/restore planning.

When `vision_auto_save` is enabled, the active `/vision` stream can automatically store detected object groups as vision memories. `vision_auto_save_interval_seconds` throttles repeated saves for the same detected object set. Operators can also save the current detection manually from the Vision page. Snapshot assets are attached only when `snapshot_storage_enabled` is enabled.

## Memory Impact

Embedding models affect:

- semantic search
- hybrid search
- context builder
- duplicate detection quality
- FAISS index dimensions

Vision models affect:

- detected object labels
- confidence scores
- vision memory content
- tags available for memory correlation

## Safe Change Checklist

Before changing model behavior:

1. Confirm whether the change is for embeddings or vision.
2. Make the change in Settings.
3. For embeddings, rebuild FAISS.
4. Test Semantic Search.
5. Test Agent Context Builder.
6. If vision changed, test `/vision` and `/api/v1/vision/status`.
7. Check System Health.
8. Confirm no secrets or camera frames are being logged.
